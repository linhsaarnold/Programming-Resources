Question 2

What is a SQL versus a NoSQL database?

SQL databses require you to add data into defined tables and fields (referred to as a "schema"). The schema contains things such as: primary keys, foreign keys, indexes, relationships, etc...

NoSQL databases allow you to add data anywhere at anytime. Perhaps more suited to projects in which the initial data requirements are harder to find and define.

Why is one preferable to another?

It depends on your project or task at hand. Some are better suited to SQL databases while toher are better for NoSQL. Sometimes it doesn't matter and either option will do.

Let's say you want to find information on how promises work in ES6. What does that mean?

ES6, aka "Harmony", is the upcoming 6th major release of ECMAScript language specification, which is the proper name for the language JavaScript. Promises are proxies for a value that will eventually become available, offering an alternative to callbacks and events. Since javascript is single threaded, it has to stop while it waits on network requestions. If it does not wait, no other code can be excuted (results in the browser stalling). If a function completes its execution but the results are unavailable until somethng happens, the function can return a "promise" object... this promises to eventually give the result you're looking for."
